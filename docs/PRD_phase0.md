# PRD — Phase 0

> 목표: 이메일 50명 수집. 돈 한 푼도 받지 않는 단계.  
> 트리거: 즉시 시작

---

## 1. 개요

에어비앤비 데이터 없이 순수 계산 로직만으로 작동하는 랜딩페이지.  
사용자가 수익성을 직접 계산해보게 만들고, 결과 화면에서 "더 정확한 데이터가 궁금하다면" 이메일 수집으로 연결.

---

## 2. 핵심 기능 (우선순위 순)

### P0 — 랜딩페이지
- 카피: "에어비앤비 창업, 어디서 시작할지 모르겠다면"
- 이메일 입력 → "출시 알림 받기" CTA 버튼
- Supabase `waitlist` 테이블에 이메일 저장

### P0 — 수익성 시뮬레이터
사용자 입력값:
- 월세 (원)
- 예상 객단가 (1박 가격, 원)
- 예상 예약률 (%)
- 초기 투자비용 (인테리어+수리비, 원)

계산 결과 출력:
- 월 예상 매출: `객단가 × 예약률 × 30일`
- 월 예상 순수익: `월매출 - 월세 - 청소비(추정) - 전기세(추정)`
- ROI: `월순수익 / 초기투자비용 × 100`
- 원금회수기간: `초기투자비용 / 월순수익` (개월)

### P0 — 창업 가계부 미리보기
수익성 시뮬레이터 결과와 연동:
- 1개월차 ~ 12개월차 가상 장부 표시
- `매출(추정) - 월세 - 청소비 - 전기세 - 세금 = 실수령액` 월별 시각화
- 수익성 시뮬레이터와 동일 페이지에서 흐름 연결

### P1 — 이메일 수집 CTA (결과 화면 하단)
- "더 정확한 내 동네 데이터가 궁금하다면 → 출시 알림 신청" 버튼
- 시뮬레이터 결과 화면 하단에 배치

---

## 3. 사용자 플로우

```
랜딩페이지 진입
→ 히어로 섹션 (카피 + 이메일 수집)
→ 스크롤 다운
→ 수익성 시뮬레이터 (입력 폼)
→ 계산 결과 + 창업 가계부 미리보기
→ "더 정확한 데이터 원해요" → 이메일 수집
```

---

## 4. 데이터 모델

```sql
-- 이메일 대기자 명단
create table waitlist (
  id uuid default gen_random_uuid() primary key,
  email text not null unique,
  created_at timestamp with time zone default now()
);
```
