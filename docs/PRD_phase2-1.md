# PRD — 지도 기반 입지 탐색 (사업계획서 3-1 구현)

> 목표: 사업계획서 **[3-1] 입지·물건 미결정 단계**를 지도 UI로 구현.
> 지도 탐색은 무료(와우 + 미끼), 핵심 분석(AirROI·종합점수)은 결제 후 잠금 해제.
> 상태: 설계 초안 (검토 필요)
> 선행: Phase 1-2 완료 (결제·리포트·AirROI 레이어 가동 중)

---

## 0. 한 줄 정의

지도 위에서 동네를 고르면 → 그 동네의 **종합 입지 점수**와 수익 데이터를 보여주는, "들어갈 동네"를 찾는 탐색 도구.

기존 주소 리포트(3-2)가 "이 건물 검증"이라면, 이 지도는 "어느 동네부터 볼지" 결정하는 **앞단**이다.

---

## 1. 핵심 설계 결정 (확정)

| 항목 | 결정 | 근거 |
|------|------|------|
| 와우포인트 | **종합 입지 점수 (0~100)** | 사업계획서 [1] "10초 입지 스코어링" |
| 핀 단위 | **영역(행정동) 단위 통계** | 개별 숙소 노출 금지(변호사) 회피 |
| 무료 범위 | 지도 + 동네 선택 + 경쟁밀도(공공) | 미끼·와우 |
| 유료 범위 | AirROI 수익·예약률 + 종합점수 | 핵심 가치, 🔒 잠금 |
| AirROI 호출 시점 | **결제 시점에만** | 비용 게이팅 |
| 비용 전략 | 지역 단위 90일 캐시 (배치 제외) | 결제 게이팅 + 재호출 0 |
| 상품 구분 | **지역 리포트 ≠ 주소 리포트** | 3-1엔 건축물대장 없음 |

---

## 2. 무료 / 유료 경계 (잠금 UX)

첨부 레퍼런스(재개발닷컴 🔒멤버십 / 오픈업 로그인 잠금 / NICE bizmap 영역 패널) 방식 차용.

### 무료 (로그인·결제 없이)
- 지도 탐색 (줌·이동)
- 동네 핀 + 이름
- **경쟁밀도** — "반경 500m 내 외도민 N개" (공공데이터, 무료)
- 종합점수·AirROI 영역은 **🔒 블러 처리 + "9,900원으로 잠금 해제"**

### 유료 (지역 결제 후)
- 해당 동네 **종합 입지 점수** (0~100, 색상)
- AirROI 동네 통계: 평균 예약률 / ADR / 월 예상 수익 / 수익 구간 분포
- 공급-수요 해석 문구 ("경쟁 많지만 예약률 높음 → 진입 가능" 등)

> 결제 단위는 **동네 1개**. 결제 후 같은 동네는 90일간 재열람 무료(캐시).

## 3. 종합 입지 점수 산식 (v1, 튜닝 대상)

지금 확보 가능한 데이터로만 구성. 역세권 점수(도보 API)는 Phase 2.

| 구성 요소 | 출처 | 방향 |
|-----------|------|------|
| 수요력 | AirROI 예약률(occupancy) | 높을수록 ↑ |
| 수익력 | AirROI ADR / revenue percentile | 높을수록 ↑ |
| 경쟁 부담 | 공공 외도민 밀도 | 높을수록 ↓ |

**핵심: 공급-수요 균형 보정** (사업계획서 [3-1] 핵심 로직)
- 경쟁 높음 + 예약률 높음 → 경쟁 페널티 완화 (검증된 수요)
- 경쟁 낮음 + 예약률 낮음 → 가점 없음 (수요 미검증)

```
score = 0.40 × 수요(예약률 정규화)
      + 0.30 × 수익(ADR percentile 정규화)
      + 0.30 × (1 − 경쟁밀도 정규화 × 수요보정계수)
```
> 가중치·정규화 기준은 실제 데이터로 캘리브레이션 후 확정. v1은 동작 우선.
> 점수는 **통계 가공값**이므로 영업비밀 이슈 없음(재확인).

색상: 초록(진입 유망) / 노랑(보통) / 빨강(레드오션) — 신호등 핀.

## 4. 지도 동작 (줌 레벨별)

| 줌 | 핀 단위 | 표시 |
|----|--------|------|
| 아웃 | 구 | "마포구 평균 78" |
| 중간 | **동** | 동별 색상 핀 (공덕동 82·연남동 75…) |
| 인 | 동 + 경쟁 | 동 점수 + 외도민 개수 |

핀 클릭 → 좌측/하단 패널 (NICE bizmap 스타일):
```
[공덕동]
경쟁밀도: 외도민 12개 (무료)
─────────────
🔒 종합 입지 점수      ← 블러
🔒 평균 예약률·ADR·수익 ← 블러
[ 9,900원으로 공덕동 정밀 분석 보기 ]
```

결제 → AirROI 호출 → 패널 잠금 해제 (+ 기존 상세 리포트로 확장 가능).

---

## 5. 비용 처리 흐름 (가장 중요)

```
[동네 선택]
  └─ 경쟁밀도(공공) 즉시 표시 — 무료, AirROI 미호출
[🔒 잠금 해제 결제]
  └─ Polar 결제 완료 웹훅
       └─ area_scores 캐시 확인
            ├─ 히트(90일 내) → 캐시 반환, AirROI 미호출 ($0)
            └─ 미스 → AirROI 1회 호출 → 점수 계산 → 캐시 저장
```
- AirROI는 **결제 웹훅 이후에만** 호출 (기존 원칙 계승).
- 동일 동네 재구매는 캐시 히트 → 비용 0.
- 일일 호출 상한(기존 100건) 그대로 적용.

## 6. 데이터 모델

### 신규: `area_scores` (지역 단위 캐시)
```sql
create table area_scores (
  id            uuid primary key default gen_random_uuid(),
  area_code     text not null,          -- 행정동 코드
  area_name     text not null,          -- '공덕동'
  center_lat    numeric(10,6) not null,
  center_lng    numeric(10,6) not null,
  minbak_count  int,                    -- 공공 외도민 수 (무료 표시용)
  airroi_data   jsonb,                  -- AirROI 응답 (결제 후 채워짐)
  score         numeric(5,2),           -- 종합 점수 (결제 후 계산됨)
  computed_at   timestamptz,
  expires_at    timestamptz             -- computed_at + 90일
);
create index on area_scores (area_code);
```

### 기존 `report_purchases` 확장
```sql
alter table report_purchases
  add column report_type text not null default 'address',  -- 'address' | 'area'
  add column area_code   text;                              -- area 타입일 때
-- address 타입은 lat/lng/address 사용, area 타입은 area_code 사용
```

> `minbak_count`는 공공데이터라 결제 전에도 채워둘 수 있음(무료 표시).
> `airroi_data`·`score`는 **첫 결제 시점에 lazy 계산**.

## 8. 검토·검증 필요 항목 (구현 전 확인)

1. **AirROI 동 단위 변별력** — `/calculator/estimate`가 같은 구 내 다른 동에서 의미 있게 다른 값을 주는지 좌표 2~3개로 사전 테스트. 변별력 없으면 핀 단위를 "구"로 하향.
2. **행정동 경계 데이터 출처** — 서울 열린데이터광장 행정동 GeoJSON 확보.
3. **파일럿 범위** — 마포 전체 동 우선? (사업계획서 Phase 1: 마포/홍대/용산/이태원)
4. **지역 리포트 가격** — 주소 리포트와 동일(9,900) vs 별도. (가격은 마지막 결정 원칙 → placeholder 유지)

## 10. 개발 순서 (Claude Code, One-at-a-time)

```
Step 1. DB — area_scores 생성 + report_purchases 확장 (RLS)
Step 2. 행정동 경계/중심좌표 적재 + minbak_count 집계 채우기
Step 3. 지도 페이지 골격 (카카오맵 + 동 핀, 무료 데이터만)
Step 4. 동 선택 패널 + 🔒 잠금 UI (블러 + CTA)
Step 5. 지역 결제 연동 (report_type='area', area_code 전달)
Step 6. 결제 웹훅 → AirROI 호출 → 점수 계산 → 캐시 → 잠금 해제
Step 7. 종합점수 산식 캘리브레이션 (실데이터 튜닝)
Step 8. 배포 + 실제 1건 결제 검증
```

> Step 1 진입 전 8번 "검증 항목" 먼저 확인 (Think-Then-Do).
