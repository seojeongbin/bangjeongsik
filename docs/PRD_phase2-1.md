# PRD — 지도 기반 입지 탐색 (Phase 2-1, 개정판 v2)

> 변경 핵심: 동 단위 "유료 결제 상품" → 동 단위 "무료 미끼 + 주소 리포트로 가는 깔때기"로 전환.
> 사유: 동 단위 유료는 (1) 공유 시 무력화 (2) 실질 입지로 안 느껴짐 (3) 재결제 유도 불가.
> 상태: 개정 (구버전 area_scores 결제 로직 폐기)
> 선행: Step 1~3 완료

## 1. 상품 구조
| 구분 | 동 지도(무료 미끼) | 주소 리포트(유료 핵심) |
|---|---|---|
| 결제 | 무료 | 9,900원 (기존 checkout 재사용) |
| 정보 | 외도민 개수 + 면적당 밀도 + AirROI 예약률 1개 지표 | 인원/방/화장실별 객단가·예약률·예상수익, 월세 대입 순수익, 종합점수(별점) |
| 공유리스크 | 낮음 | 없음(개인 주소·월세 기반) |
핵심: 동 무료에선 객단가·수익 절대 노출 안 함. 예약률만으론 판단 불가 → 결제 유도.

## 2. 무료/유료 경계
- 동 무료: 지도/핀/동이름, 외도민 개수, 면적당 밀도(경쟁밀도), AirROI 예약률 1개 지표(캐시/근사)
- 동 CTA: "이 동네 매물 실제 수익은? → 주소 입력하고 정밀 분석" → 기존 checkout(report_type 없음, 주소 기준)

## 3. 폐기 항목
- area_scores 테이블, report_purchases.report_type='area' 분기, 동 결제·90일 재열람 개념 전부 폐기
- 동 패널 별점(★)은 주소 리포트로 이동 (주소 결제 후 종합점수로 표시)

## 4. AirROI 예약률 무료노출 비용처리
- 16개 동 예약률 사전 1회 계산/적재 후 캐시 (minbak_listings식 정적 적재)
- 사용자 클릭마다 AirROI 호출 금지. "OO년 OO월 기준 추정치" 명시

## 5. 지도 UX 개선 (깔때기 연결 이후)
경계선 표시 / 줌아웃 시 개수 상시표시 / 호버 활성화 / 면적당 밀도 노출

## 6. 개발 순서
- Step A: 현재 결과물 배포
- Step B: 동 CTA → 주소입력 → 기존 checkout 연결 (신규 결제로직 0)
- Step C: 동 무료에 AirROI 예약률 1지표 (사전 캐시)
- Step D: 지도 UX 개선(5번)
- Step E: 주소 리포트 강화(인원수별 객단가 등) — 별도 PRD
구버전 Step5(area결제)/Step6(area캐시) 폐기.

## 7. 이번 개정 제외 (별도 PRD)
인테리어 프리미엄 점수 / 인원·방·화장실별 객단가 필터 / 카카오 로그인 / 웹 구조 개편 / 후기·미리보기
